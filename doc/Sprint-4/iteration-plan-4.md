## Process

### Roles & Responsibilities
- **Scrum Master:** Kazi  
  Facilitates daily standups, sprint planning, sprint review, and retrospectives. Ensures the team adheres to Agile practices and helps resolve any blockers.

- **Note Taker:** Laksh  
  Responsible for taking meeting minutes, maintaining documentation, and tracking progress updates throughout the sprint.

- **Product Owner:** Risheit  
  Manages the product backlog, prioritizes features, and ensures that team output aligns with user needs and project goals.

- **Team Members:** Aashir, Bilal, Kazi, Laksh, Risheit  
  Handle implementation, testing, reviewing each other’s code, and contributing to design and feature discussions.

---

## Events
- **Daily Standups (Weekdays, 4:00 – 4:15 PM, Online)**  
  A brief meeting to discuss daily progress, surface blockers, and plan the day.

- **RPMs (Tuesdays, 1:00 – 2:00 PM, Online)**  
  Weekly check-in with instructors/TAs to review sprint progress and receive guidance.

- **Review Meetings (Fridays, 3:00 – 5:00 PM, Online)**  
  Weekly sprint review and planning session to assess work done, demo new features, and plan for the upcoming sprint.

- **Code Reviews (Ongoing via GitHub)**  
  All code is reviewed via pull requests to ensure quality, consistency, and team alignment.

---

## Artifacts
- **Jira Board**  
  Used for sprint planning, progress tracking, and backlog management. Tasks are prioritized and assigned to team members through well-defined Jira tickets.

- **User Stories**  
  User stories guide our feature development and QA. Each ticket in Jira is mapped to one or more user stories to ensure relevance and clarity.

- **GitHub Repository**  
  Main source for all code and documentation. Pull requests are used to manage contributions and maintain code quality standards.
